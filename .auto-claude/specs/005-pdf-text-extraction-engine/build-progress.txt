=== PROJECT OVERVIEW ===

Description:
PDF processing pipeline for extracting metadata from supplier compliance certificates.
Implements automated extraction of certificate numbers, dates, issuing organizations,
company names, and certification scope from Romanian-language PDF documents.

Tech Stack:
- Runtime: Node.js + TypeScript + Express
- PDF Processing: pdf-parse library
- Database: PostgreSQL with JSONB for flexible metadata
- Date Parsing: chrono-node with Romanian locale support
- Testing: Jest (unit, integration, E2E)
- Infrastructure: Docker + docker-compose

Key Requirements:
- Extract text from text-based PDFs (95%+ success rate)
- Parse Romanian date formats (dd.MM.yyyy, dd/MM/yyyy, month names)
- Extract certificate metadata (80%+ accuracy)
- Flag failed extractions with detailed error messages
- Store results in PostgreSQL linked to document records

=== METADATA EXTRACTION PATTERNS ===

Expected Romanian certificate patterns:

Date formats:
- dd.MM.yyyy (e.g., "15.03.2024")
- dd/MM/yyyy (e.g., "15/03/2024")
- d MMMM yyyy (e.g., "15 martie 2024")
- d MMM yyyy (e.g., "15 mar 2024")

Certificate fields to extract:
- Certificate Number: "Nr. certificat:", "Certificate No:", "Certificat:"
- Issuing Organization: "SRAC", "Emis de:", "Issued by:"
- Issue Date: "Data emiterii:", "Issued on:", "Emis la:"
- Expiry Date: "Valabil până la:", "Valid until:", "Data expirării:"
- Company Name: "Denumire:", "Company:", "Firma:"
- Certification Scope: "Domeniu:", "Scope:", "Activitate:"

Romanian month names:
- ianuarie, februarie, martie, aprilie, mai, iunie
- iulie, august, septembrie, octombrie, noiembrie, decembrie
